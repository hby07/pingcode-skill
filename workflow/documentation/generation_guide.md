# 文档生成工作流

> 根据项目数据自动生成各类文档，支持多种格式。

---

## 触发条件

用户说以下任一：
- "生成文档"
- "写 PRD"
- "生成技术文档"
- "写测试报告"
- "生成 API 文档"
- "写发布说明"

---

## 工作流步骤

```
[⬜] Step 1: 选择文档类型
[⬜] Step 2: 确认数据来源
[⬜] Step 3: 生成内容
[⬜] Step 4: 预览确认
[⬜] Step 5: 保存文档
```

---

## Step 1: 选择文档类型

### 交互
```
📄 请选择要生成的文档类型：

A. 需求规格书 (PRD)
   - 产品背景、用户故事、功能列表、验收标准

B. 技术设计文档
   - 架构设计、接口设计、数据库设计、安全设计

C. API 接口文档
   - 接口列表、请求/响应格式、错误码、示例

D. 测试计划
   - 测试范围、测试策略、测试用例、资源安排

E. 测试报告
   - 测试执行情况、缺陷统计、质量评估、风险

F. 用户手册
   - 功能说明、操作指南、常见问题

G. 发布说明 (Release Notes)
   - 版本信息、新增功能、修复问题、已知问题

H. 项目周报
   - 本周进展、下周计划、风险与问题

I. 其他（请说明）

请输入选项（A-I）：___
```

---

## Step 2: 确认数据来源

### 操作
- 根据文档类型，确定需要从 PingCode 拉取的数据
- 展示数据来源列表
- 让用户确认或补充

### 交互
```
📋 文档：{文档类型}

我将从以下来源收集信息：

来源 1: PingCode 需求
- {需求1}
- {需求2}

来源 2: PingCode 任务
- {任务1}
- {任务2}

来源 3: 代码仓库
- 仓库: {仓库名}
- 分支: {分支名}

来源 4: 已有文档
- {文档1}

❓ 请问：
1. 以上范围是否正确？
2. 是否需要补充其他来源？
3. 文档输出格式？（Markdown / PDF / HTML）

[确认范围] [调整范围] [取消]
```

### 数据来源映射

| 文档类型 | 主要来源 |
|----------|----------|
| PRD | 需求工作项、用户故事 |
| 技术设计 | 需求、架构文档、代码结构 |
| API 文档 | 代码注释、Swagger |
| 测试计划 | 需求、测试用例 |
| 测试报告 | 测试执行结果、缺陷数据 |
| 用户手册 | 功能列表、UI 截图 |
| Release Notes | 版本工作项、变更记录 |
| 周报 | Sprint 数据、任务状态 |

---

## Step 3: 生成内容

### 操作
- 根据数据来源，填充文档模板
- 生成结构化内容

### 模板参考

#### PRD 模板
```markdown
# {产品名称} - 产品需求文档

## 1. 概述
### 1.1 背景
### 1.2 目标
### 1.3 范围

## 2. 用户画像
### 2.1 目标用户
### 2.2 使用场景

## 3. 功能需求
### 3.1 功能列表
### 3.2 用户故事
### 3.3 详细描述

## 4. 非功能需求
### 4.1 性能要求
### 4.2 安全要求
### 4.3 兼容性要求

## 5. 验收标准

## 6. 附录
```

#### 技术设计模板
```markdown
# {功能名称} - 技术设计文档

## 1. 概述
### 1.1 背景
### 1.2 目标

## 2. 架构设计
### 2.1 系统架构图
### 2.2 模块划分

## 3. 接口设计
### 3.1 接口列表
### 3.2 接口详情

## 4. 数据库设计
### 4.1 表结构
### 4.2 索引设计

## 5. 安全设计

## 6. 部署方案
```

#### Release Notes 模板
```markdown
# {版本号} Release Notes

📅 发布日期：{日期}

## 🚀 新增功能
- {功能1} ({关联需求})
- {功能2} ({关联需求})

## 🐛 修复问题
- {问题1} ({关联缺陷})
- {问题2} ({关联缺陷})

## ⚠️ 已知问题
- {问题描述}

## 📝 其他变更
- {变更1}

## 🔗 关联链接
- 需求文档：{链接}
- 技术文档：{链接}
```

### 交互
```

═══════════════════════════════════════════════════════════

- 总字数：{N} 字

2. 是否需要补充或修改？
```

### 操作

```

📊 保存结果：
- 文档名称：{文档名}
- 存储位置：{位置}
- 文件大小：{大小}

📝 后续建议：
1. 分享给团队成员评审
2. 收集反馈意见
3. 定期更新维护

❓ 还有其他需要吗？

[生成其他文档] [查看文档] [结束]
```

## 最佳实践

### 文档质量检查

生成后检查：
- ✅ 结构完整（章节齐全）
- ✅ 内容准确（数据来源可靠）
- ✅ 格式规范（标题层级、列表格式）
- ✅ 无遗漏（关键信息不缺失）

### 版本管理

- 每次更新文档时添加版本号
- 记录修改历史
- 保留历史版本

### 协作建议

- 重要文档先人工审核再发布
- 定期回顾和更新文档
- 建立文档评审机制
