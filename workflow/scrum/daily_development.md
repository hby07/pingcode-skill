# 每日开发工作流

> 从认领任务到完成交付的完整开发流程。

---

## 触发条件

用户说以下任一：
- "开始开发"
- "我要做任务"
- "开始写代码"
- "处理工作项"

---

## 工作流步骤

```
[⬜] Step 1: 选择任务
[⬜] Step 2: 开始任务
[⬜] Step 3: 开发过程
[⬜] Step 4: 完成任务
[⬜] Step 5: 汇报进度
```

---

## Step 1: 选择任务

### 操作
- 读取当前 Sprint 的待办任务
- 按优先级排序
- 展示给用户选择

### 交互
```
📋 当前 Sprint 有以下待办任务：

| # | 任务名称 | 优先级 | 预估工时 | 依赖 |
|---|----------|--------|----------|------|
| 1 | {任务1} | 🔴 高 | 1h | 无 |
| 2 | {任务2} | 🔴 高 | 1h | 无 |
| 3 | {任务3} | 🟡 中 | 1h | 依赖任务1 |
| 4 | {任务4} | 🟢 低 | 1h | 无 |
| 5 | {任务5} | 🟢 低 | 1h | 无 |

📊 待办：{N} 个任务，预估 {M}h

❓ 请问要开始哪个任务？（输入编号）：___
```

---

## Step 2: 开始任务

### 操作
- 更新任务状态：To Do → In Progress
- 创建 feature 分支
- 记录开始时间

### 交互
```
🚀 准备开始任务：{任务名称}

我将执行以下操作：
1. 更新任务状态：To Do → In Progress
2. 创建 feature 分支：feature/{任务ID}-{简要描述}
3. 记录开始时间

❓ 确认开始？

[确认开始] [跳过此任务] [选择其他任务]
```

---

## Step 3: 开发过程

### 操作
- Agent 辅助编码
- 定期汇报进度
- 添加工作项评论

### 交互（定时汇报）
```
💻 开发进行中...

✅ 已完成：
- {完成项1}
- {完成项2}
- {完成项3}

⏳ 进行中：
- {进行中项}

📝 我将添加以下评论到 PingCode：
─────────────────────────────────────────────────
"{任务名称} 开发进度：
 - ✅ {完成项1}
 - ✅ {完成项2}
 - ⏳ {进行中项}
 预计还需 {时间} 完成"
─────────────────────────────────────────────────

❓ 确认添加评论？

[确认添加] [修改内容] [暂不添加]
```

### 进度汇报触发条件
- 完成 30% 时
- 完成 60% 时
- 遇到阻塞时
- 用户主动要求时

---

## Step 4: 完成任务

### 操作
- 更新任务状态：In Progress → Done
- 添加完成评论
- 记录实际工时
- 关联 Git 提交

### 交互
```
✅ 任务即将完成：{任务名称}

我将执行以下操作：
1. 更新任务状态：In Progress → Done
2. 添加完成评论
3. 记录实际工时：{实际}h（预估 {预估}h）
4. 关联 Git 提交：{commit hash}

⚠️ 以上操作将写入 PingCode：
- 更新 1 个工作项状态
- 添加 1 条评论
- 更新工时记录

❓ 确认完成？

[确认完成] [标记为 Review] [取消]
```

### 状态选项
- **确认完成**：状态变为 Done
- **标记为 Review**：状态变为 In Review（等待代码评审）
- **取消**：返回上一步

---


## 阻塞处理

### 识别阻塞
- 任务依赖未完成
- 等待外部资源
- 技术难题
- 需求不明确

### 交互
```
🚧 发现阻塞！

任务：{任务名称}
阻塞原因：{原因描述}

❓ 你想怎么处理？

A. 标记为阻塞，等待解决
B. 寻找替代方案
C. 升级给负责人
D. 暂时跳过，做其他任务

请输入选项（A/B/C/D）：___
```

## Git 分支规范

### 分支命名
```
feature/{任务ID}-{简要描述}
例：feature/TASK-123-login-page

bugfix/{任务ID}-{简要描述}
例：bugfix/TASK-456-fix-validation

hotfix/{问题描述}
例：hotfix/fix-security-vulnerability
```

### 提交规范
```
{类型}({范围}): {描述}

类型：
- feat: 新功能
- fix: 修复
- docs: 文档
- style: 格式
- refactor: 重构
- test: 测试
- chore: 构建/工具

例：feat(login): 添加密码验证逻辑
```

### PR 规范
```
标题：{类型}({范围}): {描述}

描述：
- 关联任务：{任务ID}
- 变更内容：{简要描述}
- 测试情况：{测试说明}
```
