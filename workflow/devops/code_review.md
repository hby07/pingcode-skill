# 代码审查工作流

> Agent 协助进行代码审查，提供改进建议。

---

## 触发条件

用户说以下任一：
- "帮我做代码审查"
- "review 一下代码"
- "检查 PR"
- "代码评审"

---

## 工作流步骤

```
[⬜] Step 1: 选择审查对象
[⬜] Step 2: 分析代码变更
[⬜] Step 3: 提出审查意见
[⬜] Step 4: 用户确认
[⬜] Step 5: 更新评审状态
```

---

## Step 1: 选择审查对象

### 交互
```
📋 请选择要审查的对象：

A. 特定 PR（输入 PR 编号）
B. 某个分支的最新提交
C. 某个提交（输入 commit hash）
D. 某个文件的变更

请输入选项和相关信息：___
```

---

## Step 2: 分析代码变更

### 操作
- 拉取代码变更
- 分析变更内容
- 检查代码规范

### 交互
```
🔍 代码分析完成：

📊 变更概览：
- 文件数：{N} 个
- 新增行数：{+N}
- 删除行数：{-N}
- 净变更：{+/-N}

📝 变更文件：
1. {文件1} ({+/-N} 行)
2. {文件2} ({+/-N} 行)
3. {文件3} ({+/-N} 行)

⚠️ 发现以下问题：

问题 1：{问题描述}
- 文件：{文件名}
- 行号：{行号}
- 严重程度：🔴 高 / 🟡 中 / 🟢 低

问题 2：{问题描述}
- ...

❓ 请问：
1. 是否需要继续详细分析？
2. 重点关注哪些方面？
   A. 代码规范
   B. 性能问题
   C. 安全漏洞
   D. 全部

[继续分析] [查看问题详情] [跳过]
```

---

## Step 3: 提出审查意见

### 交互
```
📝 代码审查意见

═══════════════════════════════════════════════════════════

🔴 必须修复：

1. {问题1}
   - 文件：{文件名}:{行号}
   - 问题：{问题描述}
   - 建议：{修改建议}

2. {问题2}
   - ...

🟡 建议改进：

1. {问题1}
   - 文件：{文件名}:{行号}
   - 问题：{问题描述}
   - 建议：{修改建议}

🟢 可选优化：

1. {问题1}
   - ...

═══════════════════════════════════════════════════════════

📊 审查统计：
- 必须修复：{N} 个
- 建议改进：{N} 个
- 可选优化：{N} 个

❓ 请问：
1. 审查意见是否合理？
2. 是否需要修改意见？
3. 如何处理这些意见？
   A. 全部添加为 PR 评论
   B. 只添加必须修复的
   C. 手动选择
   D. 不添加评论

[确认添加] [修改意见] [取消]
```

---

## Step 4: 用户确认

### 交互
```
⚠️ 确认操作：

我将执行以下操作：
1. 向 PR #{PR编号} 添加 {N} 条审查评论
2. 更新评审状态：{状态}

❓ 确认执行？

[确认执行] [修改操作] [取消]
```

---

## Step 5: 更新评审状态

### 操作
- 添加审查评论
- 更新评审状态
- 报告结果

### 交互
```
✅ 代码审查完成！

📊 执行结果：
- ✅ 添加 {N} 条审查评论
- ✅ 更新评审状态

📝 后续建议：
1. 开发者修复问题后重新审查
2. 关注必须修复的问题
3. 考虑将建议改进纳入后续迭代

❓ 还有其他需要吗？

[审查其他代码] [查看评审记录] [结束]
```

---

## 代码审查检查清单

### 代码规范
- [ ] 命名规范（变量、函数、类）
- [ ] 代码格式（缩进、空格、换行）
- [ ] 注释完整性
- [ ] 代码可读性

### 功能正确性
- [ ] 逻辑正确
- [ ] 边界条件处理
- [ ] 异常处理
- [ ] 返回值处理

### 性能
- [ ] 无明显性能问题
- [ ] 无内存泄漏
- [ ] 无 N+1 查询
- [ ] 合理使用缓存

### 安全
- [ ] 无 SQL 注入
- [ ] 无 XSS 漏洞
- [ ] 敏感信息加密
- [ ] 权限控制正确

### 可维护性
- [ ] 代码复用
- [ ] 低耦合
- [ ] 高内聚
- [ ] 易于测试

---

## 审查意见格式

```
[严重程度] {问题类型}

{问题描述}

建议：{修改建议}

参考：{参考链接或示例}
```

### 严重程度
- 🔴 **必须修复**：会导致 bug、安全问题、严重性能问题
- 🟡 **建议改进**：代码质量、可维护性问题
- 🟢 **可选优化**：风格、命名等非关键问题

### 问题类型
- Bug：逻辑错误
- Security：安全问题
- Performance：性能问题
- Style：代码风格
- Documentation：文档问题
- Test：测试问题
