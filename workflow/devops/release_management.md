# 发布管理工作流

> 管理从版本规划到发布的完整流程。

---

## 触发条件

用户说以下任一：
- "准备发布"
- "创建版本"
- "发布新版本"
- "管理发布"

---

## 工作流步骤

```
[⬜] Step 1: 确定发布范围
[⬜] Step 2: 检查完成度
[⬜] Step 3: 生成 Release Notes
[⬜] Step 4: 创建发布版本
[⬜] Step 5: 部署验证
[⬜] Step 6: 通知相关方
```

---

## Step 1: 确定发布范围

### 交互
```
📋 确定发布范围

请选择版本号：
A. Major（主版本）：{当前+1}.0.0
B. Minor（次版本）：X.{当前+1}.0
C. Patch（补丁）：X.Y.{当前+1}
D. 自定义：___

请输入选项（A/B/C/D）：___
```

```
📋 本次发布包含的内容：

从 Sprint/迭代中选取：
| # | 需求/任务 | 类型 | 状态 |
|---|-----------|------|------|
| 1 | {需求1} | Feature | ✅ Done |
| 2 | {需求2} | Feature | ✅ Done |
| 3 | {缺陷1} | Bugfix | ✅ Done |

📊 统计：
- 新功能：{N} 个
- 缺陷修复：{N} 个
- 其他变更：{N} 个

❓ 请问：
1. 发布内容是否完整？
2. 是否需要增减内容？
3. 确认后我将检查完成度

[确认范围] [调整内容] [取消]
```

```
- ✅ Done: {N} 个

- ❌ To Do: {N} 个

测试情况：
- 用例总数：{N}
- 通过：{N} ({百分比}%)
- 失败：{N}
- 未执行：{N}

代码评审：
- 待评审 PR: {N} 个
- 已通过: {N} 个

⚠️ 问题：
1. {问题1}
2. {问题2}

❓ 是否继续发布？

A. 继续发布（忽略问题）

C. 取消发布

请输入选项（A/B/C）：___
```

## 版本号规范

### 语义化版本（SemVer）

```
MAJOR.MINOR.PATCH

MAJOR：不兼容的 API 变更
MINOR：向下兼容的功能性新增
PATCH：向下兼容的问题修正

示例：
- 1.0.0 → 1.0.1（修复 bug）
- 1.0.1 → 1.1.0（新增功能）
- 1.1.0 → 2.0.0（不兼容变更）
```

### 预发布版本

```
1.0.0-alpha.1   # 内部测试
1.0.0-beta.1    # 公开测试
1.0.0-rc.1      # 候选版本
```

## 发布检查清单

### 发布前
- [ ] 所有工作项已完成
- [ ] 所有 PR 已合并
- [ ] 测试用例全部通过
- [ ] 代码评审已完成
- [ ] 文档已更新
- [ ] Release Notes 已生成

### 发布中
- [ ] 部署脚本已测试
- [ ] 回滚方案已准备
- [ ] 监控已配置
- [ ] 告警已设置

### 发布后
- [ ] 功能验证通过
- [ ] 性能指标正常
- [ ] 无新增错误日志
- [ ] 通知已发送
